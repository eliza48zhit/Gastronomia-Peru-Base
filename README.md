🛠️ Detalles Técnicos: Escala de Sabor en Solidity

Este despliegue implementa lógica de control de entrada de datos:

Hito de la Serie: Decimocuarto bloque completado. Es el primer contrato que utiliza un uint8 con restricciones de rango (require) para clasificar la intensidad del sabor.

Escala de Picante (1-5): Lógica diseñada para estandarizar el registro de platos según su potencia (ej. Lomo Saltado: 1, Rocoto Relleno: 5).

Eficiencia y Seguridad: Optimización de tipos de datos para minimizar el uso de gas y estructura "Zero-Risk" sin funciones de transferencia de fondos.

Licencia Abierta: Cumplimiento del estándar SPDX-MIT para asegurar la transparencia del código.

📖 Modo de Uso

Consultar el Fuego: En la pestaña Read Contract, usa consultarPlato (ID 1 para el Ceviche) para ver el nivel de picante registrado.

Registrar Tradición: En la pestaña Write Contract, usa registrarPlato para añadir platos como "Aji de Gallina" (Picante: 2) o "Lomo Saltado" (Picante: 1).

Voto Inmutable: Sistema de Likes y Dislikes para medir la popularidad de la gastronomía peruana en la blockchain.
